# B端内容最佳实践

> 用途: content-writing 技能 · 参考文档
> 适用场景: 撰写生物科技B端推文和视频脚本时参考

## 一、B端 vs C端内容区别

| 维度 | B端内容 | C端内容 |
|------|---------|---------|
| 决策周期 | 数周至数月 | 即时决策 |
| 核心诉求 | ROI、风险、合规 | 体验、价格、情感 |
| 内容深度 | 技术原理+数据证据 | 场景化+感性化 |
| 目标读者 | 多人决策链（研发→采购→高管） | 个人消费者 |
| 成功指标 | 留资、询盘、下载 | 点击、购买、分享 |

## 二、B端内容黄金法则

### 2.1 金字塔法则
```
一句话结论（What）
    ↓
三行核心论据（Why）
    ↓
详细论证+数据（How）
    ↓
行动号召（What Next）
```

### 2.2 三秒法则
- 标题/封面必须让目标读者在3秒内判断"与我有关"
- 摘要/副标题需在10秒内传递核心价值
- 全文应在3分钟内读完核心内容

### 2.3 数据驱动法则
- 每一个观点至少有一个数据/案例支撑
- 数据需标注来源（增强可信度）
- "约"、"可能"等模糊词仅在预测场景使用

## 三、B端推文类型与结构

### 类型1：技术解析类（最常用）
```
问题引入 → 技术原理 → 应用方案 → 价值总结
占比：15% → 35% → 35% → 15%
```

### 类型2：行业趋势类
```
现象观察 → 数据分析 → 趋势判断 → 应对建议
占比：20% → 35% → 25% → 20%
```

### 类型3：案例分享类
```
客户背景 → 面临挑战 → 解决方案 → 量化结果 → 客户评价
占比：10% → 15% → 30% → 30% → 15%
```

### 类型4：对比评测类
```
评估维度说明 → 逐项对比 → 综合评分 → 选型建议
```

## 四、视频脚本B端特殊要求

### 4.1 信息密度
- B端视频信息密度应是C端的1.5-2倍
- 每分钟至少包含1-2个数据点
- 每30秒一个小结论

### 4.2 视觉风格
- 推荐：干净实验室风格、数据可视化、3D分子动画
- 避免：过度娱乐化、滤镜过重、与行业气质不符

### 4.3 信任元素
- 实验室/生产线实拍画面
- 认证标识/专利证书特写
- 专家/工程师出镜讲解（非演员）

## 五、常见错误

| 错误 | 纠正 |
|------|------|
| 堆砌术语不解释 | 每次引入新术语必须用一句话解释 |
| 只讲技术不讲价值 | 每段技术描述后跟一句"这意味着……" |
| 缺少CTA | 推文末尾必须有明确的下一步指引 |
| 标题党 | B端受众对夸大标题有免疫力，反而降低信任 |
| 忽略多层决策 | 内容需同时满足技术人员（要深度）和决策者（要结论） |
