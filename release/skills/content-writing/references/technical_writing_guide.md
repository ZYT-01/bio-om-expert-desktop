# 技术写作指南

> 用途: content-writing 技能 · 参考文档
> 适用场景: 将复杂生物技术概念转化为客户可理解的语言

## 一、技术概念的客户化翻译

### 1.1 翻译三步法

```
专业术语 → 功能类比 → 客户价值

示例：
"AAV衣壳的等电点差异" → "不同衣壳表面的电荷分布不同，就像磁铁的正负极"
→ "利用这个差异，我们可以像分拣不同大小的水果一样，精准分离空壳和完整病毒"
```

### 1.2 术语分级处理

| 级别 | 处理方式 | 示例 |
|------|----------|------|
| L1 必知 | 必须使用并解释 | AAV、GABA、SOD、GMP |
| L2 可替代 | 用功能描述替代 | 不写"层析柱"，写"分离柱" |
| L3 可省略 | 完全省略 | 具体缓冲液配方、仪器型号 |

### 1.3 有效类比库

| 生物技术概念 | 日常类比 |
|-------------|----------|
| 病毒载体 | "快递小哥，把治疗基因送到指定地址" |
| 空壳率 | "一堆快递盒里有多少是空的" |
| 细胞转染 | "给细胞发一封加密邮件，让它按指令生产蛋白质" |
| 色谱分离 | "像用不同网眼的筛子，把不同大小的分子分开" |
| 免疫原性 | "身体对陌生来客的警觉程度" |
| 生物利用度 | "吃进去的药，有多少真正到达了目标器官" |

## 二、数据呈现规范

### 2.1 数据翻译

| 原始数据 | 客户化表达 |
|----------|-----------|
| "空壳率从18%降至2.5%" | "每100支药剂中，从只有82支有效，提升到97支有效" |
| "收率提升3倍" | "同样的原料和时间，产出从1批变成3批" |
| "检测时间从8h到45min" | "从等一整天出结果，到午休前就能拿到报告" |

### 2.2 数据可视化原则
- 趋势用折线图，对比用柱状图，占比用饼图
- 每个图表只能有一个核心信息
- 关键数据用颜色突出（不超过2种强调色）

## 三、段落组织

### 3.1 PEEL法则
```
Point     — 段落第一句：本段核心观点
Evidence  — 数据/案例支撑
Explain   — 解释这个数据意味着什么
Link      — 回到文章主线或引出下一段
```

### 3.2 段落长度
- B端推文：每段 ≤ 300字
- 段落间用小标题或空行分隔
- 每个段落只能聚焦一个观点

## 四、检查清单

- [ ] 每个专业术语首次出现时是否解释了？
- [ ] 数据是否都标注了来源？
- [ ] 是否用了至少一个有效类比？
- [ ] 每个段落的"所以呢？"问题是否能回答？
- [ ] 技术细节是否服务于此段的客户价值？
