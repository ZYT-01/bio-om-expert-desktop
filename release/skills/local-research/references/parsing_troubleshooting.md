# 解析问题排查指南

> 用途: local-research 技能 · 参考文档
> 适用场景: 文档解析失败或结果异常时排查

## 一、常见问题速查

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| PDF 文字提取为空 | 扫描型PDF无文字层 | 启用 OCR 回退 |
| 中文乱码 | 编码检测错误 | 手动指定 GBK/GB2312 |
| Office 文档报错 | 缺少依赖库 | 安装 python-docx/openpyxl/python-pptx |
| 大文件超时 | 文件过大 | 提高 max_file_size_mb 阈值或分块处理 |
| 加密PDF | 有密码保护 | 跳过，标记为"加密文档" |
| 表格提取不完整 | 复杂合并单元格 | 标记"需人工复核" |

## 二、PDF 问题排查

### 2.1 文字层为空
```
1. 检查PDF是否由扫描仪生成 → 是 → 启用OCR
2. 检查字体是否嵌入 → 未嵌入 → 可能影响提取
3. 使用 PyMuPDF 替换 pdfplumber 重试
```

### 2.2 OCR 效果差
- 分辨率 < 200 DPI → 提高渲染分辨率
- 中英文混排 → 确认 OCR 语言包: `chi_sim+eng`
- 表格/公式 → 标记为"OCR精度有限"

## 三、Office 文档排查

### 3.1 .docx
- .doc (旧格式) → 先用 LibreOffice 转换为 .docx
- 宏文件 (.docm) → 跳过宏，仅提取文本

### 3.2 .xlsx
- 公式未计算 → 使用 `data_only=True` 读取计算值
- 合并单元格 → 仅提取左上角单元格的值

### 3.3 .pptx
- SmartArt/图表 → 无法提取，标记为"图形元素"
- 嵌入视频/音频 → 跳过，不计入文本提取

## 四、编码问题

### 4.1 编码检测顺序
1. UTF-8（优先）
2. GBK / GB2312（中文Windows环境常用）
3. BIG5（繁体中文）
4. Shift-JIS（日文）
5. Latin-1（兜底）

### 4.2 多编码混排
- 按段落分别检测编码
- 失败则使用 `errors='replace'` 替换无法识别的字符

## 五、性能优化

| 场景 | 优化方法 |
|------|----------|
| >100 个文件 | 多进程并行解析（进程数 = CPU 核数） |
| >50MB PDF | 流式读取，不一次性加载到内存 |
| 目录深度 >5 层 | 限制递归深度或分批扫描 |

## 六、错误上报格式

```json
{

  "error_type": "ocr_failed",
  "message": "OCR引擎未能启动：未安装Tesseract",

  "fallback": "仅提取嵌入的文字层（如有）"
}
```
