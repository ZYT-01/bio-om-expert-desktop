# 编码与字符集处理指南

> 用途: local-research 技能 · 参考文档
> 适用场景: 处理中文及多语种文档时的编码问题

## 一、中文字符编码背景

### 1.1 常见中文编码

| 编码 | 全称 | 适用范围 | 识别特征 |
|------|------|----------|----------|
| UTF-8 | Unicode Transformation Format - 8-bit | 全球通用 | BOM: EF BB BF |
| UTF-8-BOM | UTF-8 with BOM | Windows中文文件 | 文件头: EF BB BF |
| GBK | 国标扩展 | 简体中文Windows | 双字节，高位为1 |
| GB2312 | 国标简体中文字符集 | 旧版中文文档 | GBK子集 |
| GB18030 | 国标强制性标准 | 新版中文标准 | GBK超集 |
| BIG5 | 大五码 | 繁体中文（台湾/香港） | 双字节编码 |

### 1.2 编码检测策略

```
1. 检查BOM标记（UTF-8/UTF-16/UTF-32）
2. 使用 chardet/cchardet 库进行统计检测
3. 按置信度排序，取最高分的编码
4. 置信度 < 0.7 时，按优先级手动尝试
```

## 二、编码处理流程

```
┌──────────┐
│ 读取文件 │
└────┬─────┘
     │
     ▼
┌──────────────┐
│ 检测BOM      │───→ 有BOM ──→ 按BOM指示解码
└──────┬───────┘
       │ 无BOM
       ▼
┌──────────────┐
│ chardet检测  │───→ 置信度>0.7 ──→ 按检测结果解码
└──────┬───────┘
       │ 置信度低
       ▼
┌──────────────────────────┐
│ 按优先级依次尝试解码     │
│ UTF-8 → GBK → GB2312 →  │
│ BIG5 → Latin-1          │
└──────────────────────────┘
```

## 三、常见乱码模式与修复

### 3.1 典型乱码对照

| 乱码示例 | 原始编码 | 错误解码 | 修复方法 |
|----------|----------|----------|----------|
| `浣犲ソ` | UTF-8 | GBK | 用UTF-8重新解码 |
| `ä½ å¥½` | UTF-8 | Latin-1 | 用UTF-8重新解码 |
| `�字符` | UTF-8 | — | 缺失字符，无法修复 |

### 3.2 修复代码示例

```python
def fix_mojibake(text: str, assumed_wrong: str = "gbk", correct: str = "utf-8") -> str:
    """修复编码错乱的文本"""
    try:
        return text.encode(assumed_wrong).decode(correct)
    except (UnicodeEncodeError, UnicodeDecodeError):
        return text  # 无法修复，保持原样
```

## 四、文件编码约定

### 4.1 项目推荐编码
- 所有新创建的文件使用 **UTF-8 (无BOM)**
- Markdown/代码文件: UTF-8
- 配置文件 (JSON/YAML): UTF-8
- 报告输出: UTF-8

### 4.2 外部文件处理
- 不改变原始文件编码
- 提取文本时统一转换为 UTF-8 存储
- 在报告中标注原始编码

## 五、特殊字符处理

### 5.1 生物科技领域特殊符号
| 符号 | Unicode | HTML实体 | 说明 |
|------|---------|----------|------|
| µg | U+00B5 | `&micro;` | 微克 |
| µL | U+00B5 | `&micro;` | 微升 |
| °C | U+00B0 | `&deg;` | 摄氏度 |
| × | U+00D7 | `&times;` | 乘号 |
| α/β/γ | U+03B1-03B3 | `&alpha;/&beta;/&gamma;` | 希腊字母 |
| ™ | U+2122 | `&trade;` | 商标符号 |
| ® | U+00AE | `&reg;` | 注册商标 |

### 5.2 处理原则
- 在报告中保留原文特殊符号
- 不自动替换为ASCII等价符号
- 输出文件确保为 UTF-8 编码

## 六、跨平台注意事项

| 平台 | 默认编码 | 风险 |
|------|----------|------|
| Windows 中文版 | GBK | UTF-8文件可能显示乱码 |
| macOS | UTF-8 | 基本无问题 |
| Linux | UTF-8 | 基本无问题 |
| 老旧 Windows | GB2312 | GBK扩展字符可能丢失 |

**应对策略**：所有输出文件使用 UTF-8 BOM（`﻿`），确保 Windows 记事本能正确识别。
