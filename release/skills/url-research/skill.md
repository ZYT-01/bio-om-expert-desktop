# Skill: url-research（指定链接分析研究）

## 用途
根据指定的主题或文案，对用户提供的网址链接列表或 Excel 记录文件进行逐一分析，提取关键信息，生成结构化研究报告。

**v1.3 新增**：支持直接加载项目根目录中的 Excel 文件（如 `URL.xlsx`），自动解析多条「主题 + URL」记录并批量处理，每一条记录独立分析，最终汇总为统一的跨源研究报告。

## 触发条件

### 方式一：命令行传入 URL 列表（原有）
- 用户输入：`/url-research "主题" --urls "url1,url2,url3"`
- 示例：`/url-research "CRISPR临床进展" --urls "https://xxx.com/a,https://xxx.com/b"`

### 方式二：指定 Excel 文件（新增）
- 用户输入：`/url-research --xlsx "URL.xlsx"`
- 示例：`/url-research --xlsx "./URL.xlsx"`
- 说明：自动读取 Excel 中第一列「主题」和第二列「URL」，批量处理所有记录

### 方式三：Excel + 筛选主题（新增）
- 用户输入：`/url-research --xlsx "URL.xlsx" --topic "AAV空壳率"`
- 示例：`/url-research --xlsx "./URL.xlsx" --topic "AAV空壳率"`
- 说明：从 Excel 中筛选匹配主题的记录进行定向分析

### 方式四：Excel 指定 Sheet（新增）
- 用户输入：`/url-research --xlsx "URL.xlsx" --sheet "基因治疗"`
- 说明：加载指定 Sheet 中的记录

## 输入字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| topic | string | 条件必填 | 研究主题（方式一必填；方式二/三时优先使用 --topic 参数；未提供则使用 Excel 中记录的主题） |
| urls | array[string] | 条件必填 | 待分析的 URL 列表（方式一必填，与 --xlsx 二选一） |
| xlsx | string | 条件必填 | Excel 文件路径（方式二/三/四必填，与 --urls 二选一） |
| sheet | string | 否 | Excel Sheet 名称（方式四使用，默认第一个 Sheet） |
| context | string | 否 | 上下文说明或文案背景 |

## 执行步骤

### 步骤0：Excel 加载（xlsx 模式专有）
1. 调用 `scripts/xlsx_loader.py` 读取指定 Excel 文件
2. 自动识别列名（主题/topic/标题 → URL/链接/地址/网址）
3. 提取所有有效记录，去除空行
4. URL 标准化（自动补全 `https://` 前缀）
5. 输出结构化记录列表 + 去重 URL 列表 + 按主题分组映射

### 步骤1：链接验证
1. 调用 `scripts/link_validator.py` 检查所有 URL 有效性，记录失效链接
2. 批量模式：逐条记录对应 URL 进行验证，按主题分组汇总验证结果
3. 区分有效/失效/需权限三类状态

### 步骤2：内容抓取
1. 调用 `scripts/url_fetcher.py` 逐个（或分批）抓取 URL 内容
   - 支持动态页面（Selenium/Playwright）
   - 支持需登录的页面（需用户提供凭证）
2. 批量模式：按域名分组排队，遵守各站点限流策略
3. 每抓取完一条，记录抓取状态与耗时

### 步骤3：内容提取
1. 调用 `scripts/html_extractor.py` 提取正文、标题、关键数据
   - 识别结构化数据（表格、列表）
   - 提取引用文献和外部链接
2. 批量模式：按主题分组聚合提取结果

### 步骤4：跨源对比
1. 对多个来源进行信息对比，标记一致/矛盾之处
2. 批量模式：同一主题下的多个 URL 优先进行相互对比验证

### 步骤5：信息归纳
1. 按主题聚类，提炼核心观点
2. 批量模式：Excel 中每条记录的「主题」字段即为天然聚类标签

### 步骤6：报告生成
1. 调用 `report-generator` 技能生成 Markdown 报告
2. 批量模式：报告标题自动使用主主题，各子主题独立成章节

## 批量处理模式说明（新增）

当通过 `--xlsx` 方式加载 Excel 时，技能自动进入**批量处理模式**：

```
┌─────────────────────────────────────────────────────┐
│                  URL.xlsx 加载                       │
│  主题                    │  URL                      │
│  AAV 空壳率控制         │  https://xxx.com/a        │
│  CRISPR 临床进展        │  https://xxx.com/b        │
│  CDMO 工艺优化          │  https://xxx.com/c        │
│  ...                    │  ...                       │
└─────────────┬───────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────┐
│ 步骤1: 链接验证（全量）                              │
│   ✅ https://xxx.com/a                               │
│   ✅ https://xxx.com/b                               │
│   ❌ https://xxx.com/c (404)                         │
└─────────────┬───────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────┐
│ 步骤2-3: 逐条抓取 + 提取                             │
│   按域名分组 → 遵守限流 → 逐条处理                    │
│   每完成一条记录，输出进度摘要                        │
└─────────────┬───────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────┐
│ 步骤4-6: 跨源对比 → 归纳 → 生成报告                  │
│   同一主题下多URL互验                                 │
│   不同主题独立成章节                                  │
│   汇总统合为一份完整报告                              │
└─────────────────────────────────────────────────────┘
```

**批量处理规则**：
- 同一域名下的 URL 串行处理（间隔 2-5 秒）
- 不同域名间的 URL 可并行处理（最多 3 个并发）
- 单次批量上限 50 条记录（超出分批执行）
- 遇 403/429 自动暂停该域名，其余继续

## 输出字段

| 字段 | 类型 | 说明 |
|------|------|------|
| report_path | string | 报告文件路径 |
| total_records | int | Excel 总记录数（xlsx 模式） |
| valid_urls | array | 有效 URL 列表 |
| invalid_urls | array | 失效 URL 列表（含原因） |
| topic_groups | array | 按主题分组的分析结果 |
| conflicting_info | array | 来源间矛盾信息标记 |
| key_findings | array | 核心发现列表 |

## 依赖清单

- **scripts/**:
  - `xlsx_loader.py` — **【新增】步骤0调用，解析 Excel 文件提取主题+URL 记录**
  - `url_fetcher.py` — 步骤2调用，抓取页面内容
  - `html_extractor.py` — 步骤3调用，提取结构化信息
  - `link_validator.py` — 步骤1调用，验证链接可用性

- **assets/**:
  - `user_agents.json` — 请求头配置库，避免被屏蔽

- **templates/**:
  - `url_research_report_template.md` — URL研究报告输出骨架（v1.3 更新：支持批量多主题章节）

- **references/**:
  - `crawling_ethics.md` — 爬取合规与伦理指南
  - `rate_limit_strategy.md` — 限流策略参考，避免触发反爬
