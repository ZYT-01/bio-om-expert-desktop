# 限流策略参考

> 用途: url-research 技能 · 参考文档
> 适用场景: 配置抓取频率，避免触发反爬

## 一、默认限流配置

```json
{
  "default": {
    "requests_per_minute": 12,
    "concurrent_requests": 3,
    "min_interval_ms": 5000,
    "max_interval_ms": 10000,
    "jitter": true
  }
}
```

## 二、站点分级限流

### T0 · 高敏站点（易封IP，影响严重）
| 站点 | 请求间隔 | 日上限 |
|------|----------|--------|
| 微信公众号 | 10-15s | 50 |
| 知乎 | 5-10s | 100 |
| 微博 | 8-12s | 80 |
| 小红书 | 10-15s | 50 |

### T1 · 中敏站点
| 站点 | 请求间隔 | 日上限 |
|------|----------|--------|
| 行业媒体（生物谷等） | 3-5s | 200 |
| 新闻门户 | 2-3s | 300 |

### T2 · 低敏站点
| 站点 | 请求间隔 | 日上限 |
|------|----------|--------|
| 学术期刊（Nature等） | 5-10s | 100 |
| 预印本（BioRxiv等） | 3-5s | 200 |
| 政府/监管网站 | 3-5s | 无限制 |
| 企业官网 | 2-3s | 200 |

## 三、退避策略

### 3.1 指数退避
```
首次重试: 等待 2^1 = 2s
二次重试: 等待 2^2 = 4s
三次重试: 等待 2^3 = 8s
最大等待: 60s
```

### 3.2 响应码处理

| HTTP状态码 | 策略 |
|-------------|------|
| 200 | 正常处理 |
| 301/302 | 跟随重定向（最多3次） |
| 403 | 停止该域名，标记为"反爬阻断" |
| 404 | 标记为"失效URL" |
| 429 | 等待 Retry-After 时长后重试 |
| 5xx | 指数退避重试（最多3次） |

## 四、并发控制

```
                    ┌──────────┐
                    │ URL队列  │
                    └────┬─────┘
                         │
               ┌─────────▼─────────┐
               │   域名分组器      │
               │ 同域 → 串行队列   │
               │ 异域 → 并行队列   │
               └─────────┬─────────┘
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
     ┌─────────┐   ┌─────────┐   ┌─────────┐
     │ 域A队列 │   │ 域B队列 │   │ 域C队列 │
     │ (串行)  │   │ (串行)  │   │ (串行)  │
     └─────────┘   └─────────┘   └─────────┘
```

## 五、用户代理轮换

- 每次请求从池中随机选择 User-Agent
- 同一域名连续 3 次请求使用不同 UA
- 不使用过时/移动端 UA 抓取桌面站点
