# 报告撰写规范

> 用途: report-generator 技能 · 参考文档
> 适用场景: 报告生成时的质量和格式标准

## 一、报告命名规范

| 组件 | 格式 | 示例 |
|------|------|------|
| 日期前缀 | YYYY-MM-DD | 2026-06-26 |
| 主题 | 中文字符，≤100字符 | AAV基因治疗商业化进展 |
| 扩展名 | .md | — |
| 完整文件名 | `YYYY-MM-DD_{topic}_report.md` | `2026-06-26_AAV基因治疗商业化进展_report.md` |

## 二、内容质量标准

### 2.1 执行摘要

- 长度: 300-500 字
- 结构: 背景(1-2句) → 方法简述(1句) → 核心发现(3-5要点) → 结论与建议(1-2句)
- 独立可读: 不需阅读正文即可理解报告全部要点

### 2.2 数据引用

- 所有数据必须标注来源
- 数值需注明单位和时间
- 预测/估计数据需标注为"估算值"

### 2.3 来源标注

```
格式: [来源名称](URL) 或 `文件路径`
可信度: ⭐⭐⭐ (高) / ⭐⭐ (中) / ⭐ (低)
标注: [预印本] / [企业自宣] / [需验证] 等
```

## 三、格式规范

### 3.1 Markdown 约定

- 使用 `#` 到 `####` 标题层级，不超过4级
- 表格必须有表头行和分隔行
- 列表使用 `-` 无序符
- 代码/命令使用反引号 `` ` `` 包裹
- 使用 `> ` 块引用标注外部引用

### 3.2 表格规范

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
|     |     |     |
```

- 空表用 `—` 填充
- 数值右对齐不强制（纯 Markdown 限制）

## 四、特殊内容处理

### 4.1 敏感信息脱敏

| 场景 | 处理方式 |
|------|----------|
| 客户名称 | 替换为 `[REDACTED: 客户A]` |
| 未公开管线数据 | 替换为 `[REDACTED: 保密数据]` |
| 具体合同金额 | 替换为区间 `[约 X-Y 万元]` |

### 4.2 矛盾信息标注

```
⚠️ **信息矛盾标记**：
- 来源A: {描述} — [{名称}](URL)
- 来源B: {描述} — [{名称}](URL)
- 评估: {A更可信/B更可信/待验证}，理由: {简要说明}
```

### 4.3 时效性提示

```
> ⏱️ **时效性提示**：本节数据发布于 {日期}，已过去 {N} 个月。生物科技领域变化快速，建议核实最新进展。
```

## 五、输出检查清单

- [ ] 报告标题是否包含日期和主题？
- [ ] 执行摘要是否独立可读？
- [ ] 所有数据是否有来源标注？
- [ ] 是否有敏感信息脱敏处理？
- [ ] 是否有矛盾信息标记？
- [ ] 表格是否完整（有表头+数据）？
- [ ] 附录是否包含完整来源清单？
- [ ] 报告底部是否有生成声明？
- [ ] 文件编码是否为 UTF-8？
