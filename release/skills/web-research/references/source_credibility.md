# 来源可信度评估方法

> 用途: web-research 技能 · 参考文档
> 适用场景: 对搜索结果的来源进行可信度评分

## 一、评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 权威性 (Authority) | 30% | 发布机构的专业资质和行业地位 |
| 准确性 (Accuracy) | 25% | 信息是否可被其他来源交叉验证 |
| 时效性 (Timeliness) | 20% | 发布时间和信息所涉时间段 |
| 客观性 (Objectivity) | 15% | 是否存在利益冲突或商业动机 |
| 完整性 (Completeness) | 10% | 信息是否完整，有无选择性披露 |

## 二、可信度等级

### ⭐⭐⭐ 高可信度（≥ 80分）
可作为报告核心依据，直接引用。

**来源类型**:
- 同行评议学术期刊（Nature, Science, Cell, Lancet 等）
- 监管机构官方公告（NMPA, FDA, EMA）
- 上市公司年报/季报（经审计）
- 权威行业标准组织（ICH, USP, EP）

**验证要求**: 2 个独立来源交叉确认

### ⭐⭐ 中可信度（50-79分）
可作为辅助证据，需标注"待验证"。

**来源类型**:
- 行业专业媒体（生物谷、医药魔方、FiercePharma）
- 分析师报告（需标注机构立场）
- 学术会议海报/摘要
- 预印本（BioRxiv, MedRxiv）
- 企业新闻稿（需标注为企业自宣）

**验证要求**: 至少 1 个其他来源交叉确认

### ⭐ 低可信度（< 50分）
仅作趋势参考，不直接引用数据。

**来源类型**:
- 社交媒体帖子
- 匿名来源/爆料
- 未经验证的二手转载
- 商业推广软文

**使用规则**: 仅用于了解讨论热点，不引用为事实

## 三、特殊场景处理

### 3.1 预印本评估
- 加分: 知名研究团队、已提交同行评议
- 减分: 数据量小、无验证实验、与已发表结论冲突
- 标注: "[预印本] {标题} - 未经过同行评议"

### 3.2 企业来源评估
- 区分: 投资者关系披露（高可信度）vs 市场推广内容（低可信度）
- 关键数据需寻找第三方验证
- 标注: "[企业自宣]" 或 "[投资者关系]"

### 3.3 利益冲突检测
- 检查作者/机构的资金关系和商业关联
- 药企资助的临床研究需标注资助方
- 标注: "[资助方: {名称}]"

## 四、评分矩阵

### 权威性 (30分)
| 分数 | 标准 |
|------|------|
| 30 | 领域内公认的顶级机构/期刊 |
| 20 | 知名机构/期刊，有一定行业影响力 |
| 10 | 普通媒体/机构，可查证资质 |
| 0 | 无法确认发布者身份 |

### 准确性 (25分)
| 分数 | 标准 |
|------|------|
| 25 | 被 3+ 独立来源确认 |
| 15 | 被 1-2 个来源确认 |
| 5 | 无其他来源可验证但逻辑自洽 |
| 0 | 与其他可靠来源冲突 |

### 时效性 (20分)
| 分数 | 标准 |
|------|------|
| 20 | 6 个月内 |
| 15 | 6-12 个月 |
| 10 | 1-2 年 |
| 5 | 2-3 年 |
| 0 | 3 年以上 |

### 客观性 (15分)
| 分数 | 标准 |
|------|------|
| 15 | 无利益冲突，客观陈述 |
| 10 | 可能存在商业关联但内容平衡 |
| 5 | 有明显商业推广倾向 |
| 0 | 纯商业推广或立场极端 |

### 完整性 (10分)
| 分数 | 标准 |
|------|------|
| 10 | 信息完整，含方法/数据/来源 |
| 5 | 部分信息，有引用可追溯 |
| 0 | 碎片化信息，无明显来源 |

## 五、输出格式

```json
{
  "source": "https://xxx.com/article/123",
  "name": "来源名称",
  "type": "academic_journal",
  "scores": {
    "authority": 30,
    "accuracy": 25,
    "timeliness": 15,
    "objectivity": 10,
    "completeness": 10
  },
  "total": 90,
  "level": "high",
  "notes": "Nature Biotechnology 同行评议文章",
  "conflict_of_interest": null
}
```
